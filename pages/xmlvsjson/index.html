Diferencias

JSON

XML
JSON es ligero y fácil de leer, pero no proporciona información sobre el esquema o el tipo. Es ideal para compartir datos entre varias aplicaciones. XML es legible por los humanos pero no necesariamente por las máquinas. Puede utilizarse para compartir información estructurada entre programas y documentos.
JSON es una forma ligera y legible para el ser humano de representar estructuras de datos. XML es una forma más larga de representar datos estructurados.
JSON puede utilizarse con archivos JavaScript o de texto plano. XML sólo puede almacenarse como archivo de texto.
El formato JSON es una forma de almacenar datos de forma compacta para que los programas puedan leerlos. el formato de datos XML es una forma específica de lenguaje de marcas para almacenar datos de forma organizada.
JSON se utiliza para almacenar y transmitir datos. XML se utiliza para representar datos de forma legible por la máquina.
Puedes utilizar JSON en tu aplicación web o móvil sin preocuparte de los problemas de compatibilidad porque está ampliamente aceptado en la web y en las aplicaciones móviles. XML tiene algunos problemas de compatibilidad entre plataformas.
JSON suele ser más compacto que XML, lo que significa que puede transmitirse más rápidamente a través de las redes. JSON también tiene menos restricciones en su estructura, lo que ayuda a los programadores cuando intentan analizar grandes cantidades de datos a partir de él. XML requieren más espacio de almacenamiento que los archivos JSON (al menos si se utilizan con Node.
La sintaxis de JSON permite definir objetos fácilmente, a diferencia de la forma más verbosa con arrays o colecciones en la sintaxis de XML. Por ejemplo:

```javascript function myFunction(date) { return { "date": date }; } var obj = Object.create(null); obj["date"] = new Date(); ```

 XML es más complicada que la de JSON debido a su necesidad de referencias a entidades que pueden no ser necesarias en algunos casos (por ejemplo, si estás creando un servicio de API). XML no es legible para los humanos. Es mucho más fácil leer JSON que XML porque JSON utiliza menos caracteres, lo que facilita la comprensión del significado de los datos. JSON es más conciso. Utiliza menos caracteres para representar la misma información que XML.
En JSON, sólo se admiten como tipos de datos la cadena, el número, el booleano y el nulo. XML se pueden utilizar muchos otros tipos, como la fecha y la hora, para describir los datos de XML.
